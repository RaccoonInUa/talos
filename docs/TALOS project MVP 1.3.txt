PROJECT: Talos
Тип документа: Архітектурна Специфікація та План Розробки (PRD/TDD)
Версія: 1.0 (MVP + Product Vision)
Архітектура: "Hybrid Evolutionary Funnel" (Варіант D)
1. Executive Summary (Для всіх)
Продукт: Автономна система радіомоніторингу (100 МГц – 6 ГГц) для виявлення БПЛА,
UGV та РЕБ.
Ключова цінність: Заміна оператора-людини на AI-агента, який аналізує ефір 24/7,
використовуючи адаптивну математику (CFAR) та пошук аномалій (Autoencoder).
Користувач: Військовий або цивільний персонал без глибоких знань у радіоелектроніці.
Платформа: Embedded Linux (Orange Pi 5 / Intel N100 + Coral USB).

4. Деталізація Модулів (Для Розробників)
Тут описано 6 ключових модулів, які потрібно розробити.
Модуль 1: Input Streamer (SDR Driver)
●
Функція: Читає I/Q семпли з USB-пристрою.
●
Вимоги:
○
○
○
Має буфер (Ring Buffer), щоб не втрачати дані, поки CPU зайнятий.
Автоматичне перепідключення при втраті зв'язку з SDR (Watchdog).
Можливість перемикати частоту (Hopping) за командою від Logic.
Модуль 2: DSP Engine (Processor)
●
●
Функція: Перетворює сирі дані на метрики.
Пайплайн:
1. I/Q -> FFT (Швидке перетворення Фур'є) -> PSD (Спектральна щільність).
2. Генерація рядка "Водоспаду" (для UI та AI).
3. CFAR Block: Порівняння піків з локальним середнім.
4. Feature Extraction: Якщо CFAR спрацював, вирахувати: Center Freq,
Bandwidth, Duration, SNR.
Модуль 3: AI Worker (The Anomaly Detector)
●
●
●
●
Функція: Оцінка "нормальності" спектру.
Вхід: Зображення спектрограми (матриця, наприклад 64x64 або 128x128
пікселів).
Модель: Convolutional Autoencoder (TFLite).
Логіка:
1. Отримати матрицю.
2. Inference (Стиснути -> Розтиснути).
3. MSE (Mean Squared Error) між входом і виходом.
4. Якщо MSE > Threshold -> повернути Anomaly_
Score.
Модуль 4: Decision Logic (The Brain)
●
●
Функція: Агрегація даних та прийняття рішень.
Логіка (State Machine):
1. Отримує події від DSP (енергія) та AI (аномалія).
2. Перевіряє Whitelist (SQLite). Якщо частота є в базі - ігнорувати або
знизити пріоритет.
3. Перевіряє кореляції (Правила): "Є подія на 900МГц + аномалія на
1.2ГГц?"
.
4. Формує об'єкт Alert і пише в БД.
Модуль 5: API & Event Bus
●
●
Функція: Зв'язок світу з ядром.
Endpoints:
○
GET /status (чи працює SDR).
○
GET /alerts (список загроз).
○
WS /stream (потік водоспаду для браузера).
○
POST /calibrate (запустити режим навчання на тиші).
○

7. Ризики та Mitigation Strategies
1. 2. 3. Ризик: CPU не встигає обробляти потік 20Msps (Megasamples per second).
○
Рішення: Зменшити Sample Rate до 2-4Msps (достатньо для огляду).
Реалізувати "пропуск кадрів" (Skip Frames) для AI - аналізувати не кожен
мілісекундний зріз, а 5-10 разів на секунду.
Ризик: Autoencoder дає багато хибних спрацювань на Wi-Fi.
○
Рішення: Додати в режим "Калібрування" не тільки тишу, але й
"нормальний шум Wi-Fi" (попросити користувача увімкнути роутер під час
калібрування).
Ризик: Перегрів Orange Pi в закритому корпусі.
○
Рішення: Програмний моніторинг температури CPU. Якщо > 80°C -
зупиняти AI модуль, залишати тільки CFAR.
